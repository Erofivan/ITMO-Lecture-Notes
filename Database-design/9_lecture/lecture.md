## Patroni 

- Это open-source инстурмент для управления кластерами а автоматического переключения при сбоях для PostgreSQL
- Синхронизация данных происходит с помощью потоковой репликации
- Демон на Python, который работает на каждом усле с PostgreSQL
- Управляет жизненным циклом экзмепляра PostgreSQL
- Постоянно следит за состоянием узла и отправляет heartbeats

## Распределенное хранилище конфигурации

- Единый источник правды о состояние кластера
- Patroni поддерживает несколько etcd, Consul, ZooKeeper, Kubernetes, API

## etcd

## Терм

- Логический аналог часов в кластере
- Каждые выборы увеличивают номер терма
- В каждый момент времени в класетере может быть только один лидер для текущего терма

### Особенности

- Рандомизированный таймаут у кажого Follower
- Если последователь не слышит лидера дольше своего таймаут, он становится кандидатом, увеличивает терм, голосует за себя и просит голоса у других
- Идея рандомизации практически исключает ситуацию когда два узла одновременно становятся кандидатами и делят голоса поровну

### Правила

- Узел не может стать лижером, если его лог не содержит все закоммиченные записи из предыдущих термов. Кандидаит отправляет голосующим номер последней записи в своем логе. Голосующий не отдаст голос кандидату, чей лог отстает от его собственого
- Лидер не может коммитить записи из предыдущих термов, просто сославшись на кворум для текущей записи. Он обязан сначала записать из своего текущего терма, которая косвенно подтверит предыдущие. Эта тонкость защищает от ситуаций, когда закоммиченная запись могла бы быть утеряна при смене лидера

## Балансировищк нагрузки
- Обеспечивает единую точку входа для приложений
- Маршрутизриует запросы RW к ведущему узлу, а R запросы к ведомым
- Проверяет состояние узлов через, чтобы всего знать, кто является лидером
- Базовая балансировка (Round Robin, Least Connections)

## HAProxy

- HAProxy настраивается так, чтобы принимать трафик на разные порты
- Интеграция с Patroni
- HAProxy не анализирует SQL-запросы

## patroni.yml

- scope: Им кластера
- name: Уникальное имя узла
- restapi: Натсройки для REST API Patroni
- etcd / consul / zookepeer: Параметры подклюяения к DCS
- boostrap: Настройки для инициализации нового кластера
- postgresql: Пути к данным, параметры PostgreSQL, pg_hba.conf и postgresql.conf

## REST API

- GET /primary: возвращает 200 OK только на узле, который в данный момент является Primary
- GET /replica: возвращает 200 OK только на работающем узле с ролью Replica
- GET /patroni: возвращает JSON с подробной информацией о состоянии узла
